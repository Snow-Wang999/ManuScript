# 错误清单

> 记录项目中犯过的典型错误，以及背后的认知标签
>
> **格式**: 错误描述 → 认知标签 → 反制措施

---

## 技术决策类

### 1. 同时调试多个变量
**错误**: v0.1 开发时，同时调试 RAGFlow 和 LLM，两边都出问题时无法定位。

**认知标签**: 变量混淆

**反制措施**: 分而治之，先用 mock 数据测试一端，通过后再接入另一端。

**来源**: [READ_ME_v0_1阶段备忘](../tasks/READ_ME_v0_1阶段备忘.md)

---

### 2. 为了渐进而渐进
**错误**: 最初规划 v0.2 作为过渡版本，但其功能与 v1.0 高度重叠。

**认知标签**: 沉没成本预期 / 路径依赖

**反制措施**: 版本规划时评估"直接跳跃"的可行性，不要为了"渐进"而创造不必要的中间步骤。

**来源**: [READ_ME_v0_2阶段备忘](../tasks/READ_ME_v0_2阶段备忘.md)

---

### 3. 所有代码放一个文件
**错误**: v1.0 初期想把所有 Agent 放在一个文件里，导致难以阅读、修改和测试。

**认知标签**: 过早优化的反面 - 过度简化

**反制措施**: 模块化不是过度设计，是必要的工程实践。一个 Agent 一个文件。

**来源**: [READ_ME_v1_0阶段备忘](../tasks/READ_ME_v1_0阶段备忘.md)

---

## 工程实践类

### 4. 跳过单独测试直接集成
**错误**: 如果跳过单个 Agent 测试直接集成，出错时无法定位是哪个 Agent 的问题。

**认知标签**: 调试成本低估

**反制措施**: 开发顺序：写代码 → 单独测试 → 测试通过 → 下一个 → 全部通过后再集成。

**来源**: [READ_ME_v1_0阶段备忘](../tasks/READ_ME_v1_0阶段备忘.md)

---

## 环境配置类

### 5. Windows 编码问题
**错误**: Python 在 Windows 上默认使用 GBK 编码，中文字符串在日志输出时报错。

**认知标签**: 环境假设

**反制措施**: 所有 .py 文件开头加 `# -*- coding: utf-8 -*-`，日志和错误信息用英文。

**来源**: [READ_ME_v0_1阶段备忘](../tasks/READ_ME_v0_1阶段备忘.md)
