# 认知标签库

> 收集项目中识别出的思维模型和认知偏误
>
> 这些标签帮助快速识别问题类型，找到对应的"解药"

---

## 已识别的认知标签

### 变量混淆
**定义**: 同时改变多个变量，导致无法判断哪个变量导致了结果变化。

**典型场景**: 同时调试两个系统、同时修改多处代码

**解药**: 控制变量法，一次只改一个

**项目案例**: v0.1 同时调试 RAGFlow 和 LLM

---

### 沉没成本预期
**定义**: 因为已经规划了某个步骤，即使发现不必要也继续执行。

**典型场景**: 版本规划、功能设计

**解药**: 定期重新评估计划的必要性，允许跳过

**项目案例**: v0.2 被跳过

---

### 路径依赖
**定义**: 因为"一直这样做"而继续这样做，不考虑是否有更好的路径。

**典型场景**: 技术选型、流程设计

**解药**: 定期问"如果从零开始，我还会这样选吗？"

---

### 调试成本低估
**定义**: 低估了集成后调试的难度，跳过单元测试。

**典型场景**: 多模块集成、Agent 编排

**解药**: 先测试再集成，永远不要跳过

**项目案例**: v1.0 Agent 开发

---

### 环境假设
**定义**: 假设代码在所有环境下行为一致。

**典型场景**: 跨平台开发、编码问题

**解药**: 明确列出环境依赖，在目标环境测试

**项目案例**: Windows GBK 编码问题

---

### 过度简化
**定义**: 为了"简单"而把不该合并的东西合并。

**典型场景**: 代码组织、架构设计

**解药**: 区分"简单"和"简陋"，模块化是必要的

**项目案例**: 所有 Agent 放一个文件
