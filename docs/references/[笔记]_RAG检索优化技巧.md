根据提供的文档内容，RAG（检索增强生成）检索相关的技巧可以分为以下几个方面：
# 一、检索前优化

## 查询优化：

- **Extract-Refine-Retrieve-Read (ERRR)**：首先从LLM中提取参数化知识，然后使用专门的查询优化器细化查询，以确保只检索到生成准确答案所需的最相关信息。
- **查询改写/重写**：在检索前改写用户查询，以提高与知识库的匹配度。
- **生成伪响应/查询扩展**：先生成问题的伪响应，然后比较这些伪响应与文档的相似性进行检索；或将查询扩展为多个相关查询。
- **多查询/链式验证**：通过多轮查询或验证机制来明确信息需求。

## 自适应检索决策：

- **Context Awareness Gate (CAG)**：动态判断用户查询是否需要外部检索。通过计算查询与上下文的相似度分布，决定是否启用RAG流程，避免无关检索损害模型表现。
- **Self-RAG**：引入批判模型，选择性决定何时进行检索。

# 二、检索方法与架构

## 检索方法：

- **稀疏检索**：如TF-IDF、BM25，基于关键词匹配，简单直观，但在处理同义词方面有局限。
- **稠密检索**：使用嵌入模型（如对比学习训练的编码器）将查询和文档转换为向量，通过向量相似度（如余弦相似度）进行语义匹配。这是当前主流方法。
- **混合检索**：结合稀疏检索和稠密检索的优点，通常能取得更好效果，但需平衡计算成本。

## 检索架构演进：

- **传统RAG/VectorRAG**：基于向量数据库的语义相似性检索。
- **GraphRAG / 知识图谱增强检索**：将文档构建为知识图谱，利用实体、关系进行结构化检索，特别适合处理复杂、多跳的推理查询，能更好地捕捉长距离关联。例如MedGraphRAG（医学领域）、Think-on-Graph 2.0、LightRAG（集成图索引的双层检索）。
- **HybridRAG**：结合VectorRAG和GraphRAG，从向量库和知识图谱中同时检索上下文，在特定领域（如金融文档）中表现更优。
- **多模态检索**：用于处理视觉输入或其他模态信息，通过“检索即生成”策略提升多模态内容生成的问答能力和速度。
- **Cascading Retrieval（如Pistis-RAG）**：采用多级（匹配、排序、重排、推理、聚合）级联检索框架，逐步优化检索结果。

# 三、检索后处理与优化

## 重排序：

- **必要性**：文档指出，没有重排模块会导致性能显著下降。
- **方法**：使用如MonoT5等模型对检索到的文档根据其与查询的相关性进行精细排序，可以有效提高生成答案的质量。

## 结果压缩与过滤：

- **ChunkRAG**：在文档块级别进行过滤。使用语义分块将文档划分为连贯的片段，并利用LLM对每个块进行相关性评分，在生成前过滤掉不相关的块，以减少幻觉并提高事实准确性。
- **信息再打包**：Reverse配置（将更相关的上下文放在离查询更近的位置）被证明能带来更优结果。合理组织信息以消除冗余，供生成模块使用。

# 四、针对效率和性能的权衡策略

- **实验推荐策略**：通过广泛实验，论文《Searching for Best Practices in Retrieval-Augmented Generation》提出了两种部署方案：

    - 追求极致性能的实施方案。
    - 平衡效率与效能的实施方案。

## 具体建议：

- **检索模块**：虽然“Hybrid with HyDE”方法RAG分数最高（0.58），但计算成本高（11.71秒/查询）。因此，推荐使用“Hybrid”或“Original”方法，它们在保持可比性能的同时降低了延迟。
- **块大小与数量**：增加块的数量和大小可以提高召回率，但效果在相关信息总量固定后会饱和。平衡成本与性能，块无需过大。在上下文数量有限的情况下，更少但更大的块有利于提高上下文精度；但若要减少噪声敏感度，则需更多更小的块以稀释噪声影响。

# 五、针对特定场景的优化

- **领域特定优化**：如医疗领域的MedGraphRAG，使用混合静态-语义分块和三层分层图结构进行精确检索。
- **处理半结构化数据**：如网络数据，传统RAG可能不佳，需要新的方法（如FastRAG）来应对。
- **处理模糊信息需求与未结构化知识**：现有方法存在局限，需要MEMORAG等新一代RAG通过记忆启发的知识发现来改进。

# 六、评估与诊断

- **评估指标**：可使用RAGAs等基准建议的指标进行评估，如忠实度、上下文相关性、答案相关性、答案正确性，以及计算检索文档与标准文档之间的余弦相似度作为检索相似性指标。
- **检索精度评估**：对于RAG模块，可以使用P@n（如前3的精度，P@3）来衡量检索文本与问题的相关性。

**总结**：RAG检索的核心技巧在于检索前对查询的智能优化、采用更先进的检索架构（如图检索、混合检索）、检索后对结果进行重排和精炼，以及根据具体应用场景在效率与性能间做出针对性权衡。同时，建立细粒度的评估体系对诊断和提升RAG系统至关重要。
