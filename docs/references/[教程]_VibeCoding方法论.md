# Vibe Coding 技巧与原则指南

> 基于 ai-guide-main 教程提炼的核心技巧，帮助你更好地用 AI 做项目

---

## 一、什么是 Vibe Coding

**Vibe Coding = 用人话和 AI 聊天 + AI 帮你写代码 + 你们一起迭代优化**

核心理念：**意图驱动编程**
- 你负责 "想清楚要做什么"（表达意图）
- AI 负责 "把它做出来"（实现逻辑）
- 你们一起迭代优化（协作进化）

关键转变：从 "怎么做"（How）→ "做什么"（What）

## 二、五大核心心法

### 心法一：Planning is Everything（规划至上）

**花在规划上的 5 分钟，能省下返工上的 30 分钟。**

规划前要回答的问题：
- 这个应用的核心功能是什么？
- 用户会怎么使用它？
- 哪些功能是必须的，哪些可以以后再加？
- 有没有什么特殊的限制或要求？

实践技巧：
- 让 AI 扮演产品经理，帮你理清需求
- 使用 AI 工具的 Plan Mode（规划模式）
- 先确定整体结构和方向，再动手写代码

### 心法二：MVP 思维（最小可行产品）

**先做一个最简单、但能用的版本，然后再慢慢完善。**

MVP 思维的好处：
1. 降低难度：只需解决最重要的问题
2. 快速验证：验证想法是否可行
3. 保持动力：看到作品成长更有成就感
4. 方便调整：发现方向不对能快速调整

实践方法：
```
告诉 AI："我们先做一个 MVP 版本，只包含最核心的 2-3 个功能。"
```

**记住：完成比完美更重要。**

### 心法三：迭代优于完美

**不要期待一次就做对，要通过多次迭代来逐步接近目标。**

迭代的节奏：
1. 提出需求 → 2. AI 生成代码 → 3. 测试运行 → 4. 发现问题 → 5. 调整优化 → 重复

任务拆分示例（登录页面）：
1. 先做简单的登录表单（只有邮箱和密码）
2. 加上表单验证
3. 连接后端 API
4. 加上错误提示和加载动画

### 心法四：上下文是王道

**AI 没有记忆。每次新对话，都要提供足够的背景信息。**

需要提供的上下文：
- 项目用的技术栈
- 已完成的功能
- 设计风格和规范
- 当前要解决的问题

实践技巧：
- 使用 CLAUDE.md 或 .cursorrules 文件
- 每次新对话先说明技术背景
- 引用已有代码让 AI 保持一致性

### 心法五：像产品经理一样思考

**你是产品经理，AI 是你的开发团队。**

产品经理思维要点：
- 理解用户的真正需求
- 把需求拆解成清晰的功能点
- 考虑用户体验的每一个细节
- 在功能、时间、质量之间做权衡

需求描述对比：
```
❌ "做一个好看的按钮"
✅ "做一个圆角、蓝色背景、白色文字、鼠标悬停时变深蓝色的按钮"
```

## 三、5 步项目开发流程

### 步骤 1：需求研究（Research）

创建 `RESEARCH.md`，记录：
- 目标：想解决什么问题
- 调研发现：类似产品的优缺点
- 核心需求：最重要的功能列表

### 步骤 2：产品需求文档（PRD）

让 AI 帮你扩展需求为完整的 PRD：
```
请帮我把这个需求扩展成完整的产品需求文档（PRD），要包含：
1. 产品概述和目标用户
2. 详细的功能列表
3. 功能优先级（MVP 和后续版本）
4. 界面设计要求
5. 技术栈建议
```

### 步骤 3：技术设计文档（Tech Design）

创建 `TECH_DESIGN.md`，包含：
- 技术栈选择
- 项目结构
- 数据模型
- 关键技术点

### 步骤 4：AI 代理指令（AGENTS.md）

创建给 AI 看的规则文件：
```markdown
# 项目开发指令

## 项目概述
[简要描述]

## 开发规范
- 使用 TypeScript，确保类型安全
- 组件使用函数式组件 + Hooks
- 使用 Tailwind CSS 编写样式

## 代码风格
- 组件名使用 PascalCase
- 函数名使用 camelCase

## 注意事项
- 保持代码简洁，避免过度设计
- 优先实现核心功能
```

### 步骤 5：实现和迭代（Build）

三步走策略：
1. **生成基础框架**：先让项目能跑起来
2. **逐步实现核心功能**：一个一个功能来
3. **优化实现细节**：打磨用户体验

## 四、对话工程技巧

### 核心原则

用 **对话思维** 代替 **提示词思维**：
- 提示词思维：一次性想把所有需求说清楚
- 对话思维：逐步明确需求，边聊边想

### 六大技巧

1. **从大到小，逐步细化**
   - 先确定整体方向
   - 再深入具体细节

2. **具体而非抽象**
   ```
   ❌ "加一个用户友好的错误提示"
   ✅ "当邮箱格式不对时，在输入框下方显示红色文字'请输入有效的邮箱地址'"
   ```

3. **提供参考和示例**
   - 用文字描述参考网站
   - 直接上传截图让 AI 参考

4. **分步骤提问**
   - 复杂功能拆成多个步骤
   - 每完成一步测试后再继续

5. **用提问引导思考**
   ```
   "这个页面加载有点慢，你觉得可能是什么原因？有什么优化建议？"
   ```

6. **让 AI 主动追问你**
   ```
   "我想做【你的需求】。请你在回答前，先问我几个问题，了解更多细节。"
   ```

### 需求描述框架（5 要素）

1. **What**（是什么）：要做什么功能？
2. **Why**（为什么）：功能的目的是什么？
3. **How**（怎么做）：技术实现有什么要求？
4. **Style**（什么样）：外观和交互要求？
5. **Where**（什么情况）：在什么场景下使用？

## 五、上下文管理技巧

### 上下文的三个层次

1. **项目级**：技术栈、设计规范、目录结构
2. **功能级**：当前功能的需求和依赖
3. **对话级**：当前对话中的临时信息

### 核心文档

| 文件 | 作用 | 内容 |
|------|------|------|
| README.md | 项目身份证 | 技术栈、目录结构、开发规范 |
| TODO.md | 任务清单 | 已完成、进行中、待开发、已知问题 |
| AGENTS.md | AI 规则 | 开发规范、代码风格、注意事项 |

### 短期记忆管理

- **一个对话一个任务**：不要混杂多个功能
- **定期压缩上下文**：让 AI 总结进展后开新对话
- **善用回顾**：新对话开始时简要回顾之前内容

### 上下文断裂的修复

识别信号：
- AI 用了不同的技术栈
- 忘记之前的设计方案
- 代码风格不一致

修复方法：
1. 重新提供上下文
2. 引用之前的内容
3. 开新对话重新开始
4. 明确暂停，让 AI 确认目标

## 六、避免 AI 失控的技巧

### 1. 项目模块化
把大项目分割成小模块，减少 AI 的上下文负担

### 2. 限定修改范围
```
仅修改 components/AddTransaction.tsx 文件：
1. 添加表单验证功能
2. 保持现有的样式和布局
3. 不要改动其他文件
```

### 3. 抽象和复用
```
请帮我抽象这个页面为可复用的组件，让其他页面可以通过传入不同的参数来使用。
```

### 4. 版本控制（Git）
每完成一个功能就提交：
```bash
git add .
git commit -m "实现添加记录功能"
```

### 5. 人工控制介入
当 AI 陷入循环时：
- 手动指定上下文
- 换个角度描述问题
- 清空对话重新开始
- 手动修改部分代码给 AI 正确方向

### 6. 多元 AI 协作
用一个 AI 分析问题，用另一个 AI 实现解决方案

## 七、常见陷阱

| 陷阱 | 解决方法 |
|------|----------|
| 一次问太多 | 一次只问一个功能 |
| 假设 AI 有记忆 | 引用之前内容或重新说明 |
| 不测试就继续 | 每次生成代码后立即测试 |
| 过度依赖 AI | 简单问题自己查文档更快 |
| 需求描述模糊 | 用具体、明确的语言描述 |
| 跳过规划 | 先想清楚再动手 |

## 八、实用 Prompt 模板

### 开始新功能
```
我要开发一个新功能：【功能描述】
技术栈：【技术栈】
请帮我：
1. 分析核心需求
2. 建议实现方案
3. 列出可能遇到的问题
```

### 调试问题
```
问题：【问题描述】
报错：【错误信息】
代码：【代码片段】
请帮我：
1. 分析可能的原因
2. 给出解决方案
3. 解释为什么会出现这个问题
```

### 优化代码
```
代码：【代码片段】
功能：【功能说明】
请帮我：
1. 找出性能问题
2. 改进可读性
3. 指出潜在 bug
```

## 九、核心原则速记

```
规划 > 编码
完成 > 完美
MVP > 全功能
迭代 > 一次完成
上下文 > 隐含假设
产品思维 > 技术思维
具体描述 > 抽象表达
小步前进 > 大步跨越
```

---

## 十、开始你的项目

1. 明确你要做什么（需求研究）
2. 写清楚功能列表（PRD）
3. 确定技术方案（Tech Design）
4. 创建 AI 规则文件（AGENTS.md）
5. 小步迭代，逐步实现
6. 及时测试，及时提交

**记住：Vibe Coding 不只是让 AI 写代码，更重要的是你如何引导 AI、如何管理整个开发过程。**
