# v0.2 阶段备忘

> **版本**: v0.2 基础流程
> **状态**: ⏸️ 已跳过
> **决策**: 见 ADR-07

---

## 本阶段原定目标

在 v0.1 基础上增加：
1. JSON 格式的大纲管理
2. Query Generator（从大纲生成检索查询）
3. 2步 Prompt Chain（检索 → 生成）
4. 引用格式化

---

## 为什么跳过这个版本

**核心原因**：v0.2 的功能与 v1.0 高度重叠。

| v0.2 计划功能 | v1.0 对应实现 |
|--------------|--------------|
| Query Generator | Query Generator Agent |
| 引用格式化 | Writer Agent 内置 |
| 2步 Prompt Chain | 6 Agent 链式调用 |

继续做 v0.2 意味着：
- 写一遍"简化版"代码
- 再写一遍"完整版"代码
- 两套代码需要分别维护

**决策**：直接跳到 v1.0，用 Agent 架构一步到位。

---

## 教训总结

### 版本规划要看全局

最初的版本规划是线性的：v0.1 → v0.2 → v1.0 → v2.0

但实际开发中发现，v0.2 是个"过渡版本"，它的存在只是为了"渐进式"引入复杂度。

**问题**：如果 v1.0 的复杂度可以直接驾驭，v0.2 就是浪费时间。

**认知标签**：避免"为了渐进而渐进"，评估直接跳跃的可行性

### 文档先行的价值

正是因为在动手前写了 v0.2 的任务文档，才发现它和 v1.0 的重叠。

如果直接开始写代码，可能写到一半才发现问题。

---

## 保留的价值

虽然 v0.2 被跳过，但它的设计思路被保留：

1. **JSON 大纲** → v1.0/v2.0 的输入格式
2. **Query Generator** → v1.0 的独立 Agent
3. **引用格式化** → Writer Agent 的内置功能
