## 教训总结

### 1. 先 LLM 后 RAG

最初想同时调试 RAGFlow 和 LLM，结果两边都出问题时无法定位。后来改为：
1. 先用 mock 数据测试 LLM 生成
2. LLM 通过后再接入 RAGFlow

**认知标签**：分而治之，隔离变量

### 2. Windows 编码是个坑

Python 在 Windows 上默认使用 GBK 编码，中文字符串在日志输出时会报错。

**解决方案**：所有 .py 文件开头加 `# -*- coding: utf-8 -*-`，日志和错误信息用英文。

### 3. 不要过早优化

v0.1 的代码很"丑"，没有抽象，没有复用。但这是对的——先跑通，再优化。
