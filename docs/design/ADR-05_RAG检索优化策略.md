# RAG 检索架构优化决策

> 基于 ManuScript 项目特点的 RAG 技术选型决策

---

## 项目约束条件

| 约束 | 说明 |
|-----|------|
| **核心目标** | 学术论文写作，生成带准确引用的段落 |
| **最高优先级** | 引用准确性 |
| **技术栈** | RAGFlow API（托管服务） |
| **开发原则** | MVP优先，不要过度设计 |

---

## 核心优先级（MVP 必做）

### 1. 重排序（Reranking）

**决策依据**：文献明确指出"没有重排模块会导致性能显著下降"

**实施方案**：
- 使用 MonoT5 或类似模型对 RAGFlow 返回结果进行二次排序
- 对检索到的文档根据与查询的相关性进行精细排序

**预期效果**：有效提高生成答案的质量和引用准确性

---

### 2. 混合检索（Hybrid Retrieval）

**决策依据**：
- 文献推荐 "Hybrid" 方法，在保持可比性能的同时降低延迟
- 学术文献常有专业术语，纯语义检索可能遗漏关键词匹配
- "Hybrid with HyDE" 虽然 RAG 分数最高（0.58），但计算成本高（11.71秒/查询）

**实施方案**：
- 结合稀疏检索（BM25）和稠密检索的优点
- 确认 RAGFlow 配置已启用混合检索模式

**预期效果**：平衡检索精度与响应速度

---

### 3. ChunkRAG 级别过滤

**决策依据**：
- 在文档块级别进行相关性评分和过滤
- 减少幻觉并提高事实准确性
- 直接支持"引用准确性最高优先级"的要求

**实施方案**：
- 使用语义分块将文档划分为连贯片段
- 利用 LLM 对每个块进行相关性评分
- 在生成前过滤掉不相关的块

**预期效果**：减少噪声输入，提高引用的精确性

---

## 次优先级（V1 阶段考虑）

### 4. 查询改写（Query Rewriting）

**应用场景**：用户输入的章节标题可能不够精确

**方案**：在检索前改写用户查询，提高与知识库的匹配度

---

### 5. Reverse 信息打包

**决策依据**：Reverse 配置（将更相关的上下文放在离查询更近的位置）被证明能带来更优结果

**方案**：合理组织检索结果顺序，消除冗余

---

### 6. 块大小调优

**决策依据**：
- 在上下文数量有限的情况下，"更少但更大的块"有利于提高上下文精度
- 增加块的数量和大小可以提高召回率，但效果会饱和

**方案**：根据实际测试结果调整 chunk_size 参数

---

## 暂不采用的技术

| 技术 | 排除原因 |
|-----|---------|
| **GraphRAG / 知识图谱** | 复杂度高，MVP 阶段属于过度设计 |
| **多模态检索** | 当前场景是 PDF 文献，暂不涉及图像等模态 |
| **自适应检索决策（CAG/Self-RAG）** | 增加系统复杂度，MVP 阶段不需要 |
| **HyDE（假设文档嵌入）** | 计算成本高，性价比不足 |

## 实施路线图

```
┌─────────────────────────────────────────────────────────────┐
│  MVP 阶段                                                    │
│  RAGFlow 混合检索 → 重排序 → 生成                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  V1 阶段                                                     │
│  + 查询改写 + ChunkRAG 过滤 + Reverse 打包                   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  V2 阶段（可选）                                             │
│  + 块大小动态调优 + 更复杂的检索策略                          │
└─────────────────────────────────────────────────────────────┘
```

## 评估指标

参考文献建议的评估方式：

| 指标 | 说明 |
|-----|------|
| **P@n（如 P@3）** | 检索文本与问题的相关性 |
| **忠实度** | 生成内容与检索内容的一致性 |
| **上下文相关性** | 检索内容与查询的匹配度 |
| **答案正确性** | 最终输出的准确性 |
| **检索相似度** | 检索文档与标准文档的余弦相似度 |
