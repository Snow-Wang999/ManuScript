## 关键决策详解

### 1. 项目定位：素材组装 vs Deep Research

**决策**：ManuScript 是**封闭域素材组装系统**，不是 Deep Research。

```
ManuScript 定位矩阵：

                    开放域（互联网）
                         ↑
    OpenAI Deep Research │  Anthropic Research
                         │
   ←─────────────────────┼─────────────────────→
   单点检索              │              深度综合
                         │
                         │  ★ ManuScript v2.0
        SciSpace         │    (封闭域素材组装)
                         ↓
                    封闭域（用户文献库）
```

**核心差异**：

| 维度 | Deep Research | ManuScript |
|------|--------------|------------|
| 信息源 | 开放互联网 | 用户已审核的文献库 |
| 幻觉风险 | 高（30-70%） | 极低（<5%） |
| 用户控制 | 低（黑盒） | 高（全程可见） |
| 适用场景 | 探索性调研 | 学术写作 |

> 详见 [ADR-01](ADR-01_v2.0整体架构与项目定位.md)

---

### 2. 技术栈选择：三种架构对比

**决策**：选择**架构 B（LangGraph + RAGFlow API）**。

| 架构 | 技术栈 | 适合场景 | 选择理由 |
|------|-------|---------|---------|
| A | Dify + RAGFlow | 快速验证 v0.1 | 灵活度受限 |
| **B** | **LangGraph + RAGFlow** | **v2.0 复杂逻辑** | **状态管理强，可编排** |
| C | Claude SDK (MCP) | 本地隐私场景 | 受限于本地算力 |

> 详见 [ADR-03](ADR-03_三种架构路线对比.md)

---

### 3. RAG 优化策略：MVP 阶段聚焦

**决策**：MVP 阶段只做**混合检索 + 重排序**。

```
MVP 阶段（必做）：
  RAGFlow 混合检索 → 重排序 → 生成

V1 阶段（考虑）：
  + 查询改写 + ChunkRAG 过滤

暂不采用：
  × GraphRAG（复杂度高）
  × HyDE（计算成本高）
  × 多模态检索（场景不需要）
```

> 详见 [ADR-05](ADR-05_RAG检索优化策略.md)

---

### 4. 版本跳跃：v0.1 → v1.0

**决策**：跳过 v0.2，直接从 v0.1 到 v1.0。

**原因**：
1. v0.2 的 Query 生成、引用格式化在 v1.0 已有 Agent 实现
2. v0.1 已验证核心假设（RAGFlow 可用、LLM 质量可接受）
3. 避免重复开发，直接验证 Agent 架构

> 详见 [ADR-07](ADR-07_跳过v0.2版本决策.md)
