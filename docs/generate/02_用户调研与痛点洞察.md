```
┌─────────────────────────────────────────────────────────────────────────┐
│                        学术写作典型流程                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   阶段 1: 文献收集 (1-4 周)                                             │
│   ├── 检索数据库（Google Scholar, Web of Science, CNKI）               │
│   ├── 下载 PDF，导入 Zotero/EndNote                                    │
│   └── 初步筛选，标记重要文献                                           │
│                                                                         │
│   阶段 2: 文献阅读 (2-8 周)                                             │
│   ├── 精读核心文献，做笔记                                             │
│   ├── 泛读参考文献，了解背景                                           │
│   └── 整理思维导图/文献矩阵                                            │
│                                                                         │
│   阶段 3: 大纲规划 (3-7 天)                                             │
│   ├── 确定论文结构                                                     │
│   ├── 分配各章节内容                                                   │
│   └── 与导师讨论确认                                                   │
│                                                                         │
│   阶段 4: 初稿写作 (2-8 周) ← 【ManuScript 聚焦阶段】                   │
│   ├── 面对空白文档发呆                                                 │
│   ├── 翻找之前的笔记和文献                                             │
│   ├── 写一段，查一段，引用一段                                         │
│   └── 反复修改，进度缓慢                                               │
│                                                                         │
│   阶段 5: 修改完善 (2-4 周)                                             │
│   ├── 导师/合作者反馈                                                  │
│   ├── 语言润色                                                         │
│   └── 格式调整                                                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 初稿写作阶段的行为观察

通过访谈和自我体验，观察到以下典型行为模式：

| 行为 | 频率 | 背后的需求/痛点 |
|------|------|----------------|
| **反复切换窗口** | 极高 | 在 Word、PDF、笔记软件之间切换，寻找相关内容 |
| **重新阅读已读过的文献** | 高 | 忘记了之前的笔记，需要重新确认 |
| **写了删、删了写** | 高 | 不确定表述是否准确，不敢下笔 |
| **搜索"XXX 怎么引用"** | 中 | 引用格式焦虑 |
| **长时间盯着空白文档** | 中 | 结构性卡顿，不知道如何开始 |
| **先写容易的部分** | 中 | 回避难写的章节 |
| **咨询同学/学长** | 中 | 寻求经验和模板 |

### 3.3 用户的 "Jobs-to-be-Done"

| Job Statement | 核心诉求 |
|---------------|---------|
| "帮我把 50 篇论文里**这一节需要的内容**提取出来" | 降低认知负荷 |
| "让我**确信每句话都有出处**，不用担心引用问题" | 消除引用焦虑 |
| "给我一个**可以修改的起点**，而不是空白文档" | 打破写作卡顿 |
| "让我**专注于思考和修改**，而不是机械的整理工作" | 提升效率 |

## 4. 核心痛点深度剖析

### 4.1 痛点 1：认知过载 (Cognitive Overload)

#### 现象描述

```
用户的真实声音：

"我读了 50 篇论文，做了几万字的笔记，但写的时候脑子一片空白。"

"读论文的时候觉得都很重要，但写到某一节的时候，
 不知道应该引用哪几篇，又要重新翻一遍。"

"感觉自己像个人肉数据库，但检索功能坏了。"
```

#### 心理机制分析

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     认知过载的心理机制                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   工作记忆限制（Miller's Law: 7±2）                                     │
│        │                                                                │
│        ▼                                                                │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │  读论文 A → 记住要点 → 读论文 B → 记住要点 → ... → 读论文 N     │  │
│   │                                                                 │  │
│   │  当 N > 7 时，早期信息开始被"挤出"工作记忆                      │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│        │                                                                │
│        ▼                                                                │
│   写作时需要"召回"所有相关信息 → 失败 → 焦虑 → 效率下降               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### ManuScript 解决方案

| 问题 | 解决方案 | 原理 |
|------|---------|------|
| 信息过载 | **Section Context Window** | 只展示与当前章节相关的 5-10 个片段 |
| 记忆失效 | **自动素材召回** | 系统代替人脑做"检索" |
| 注意力分散 | **聚焦单一章节** | 每次只处理一个 Section |

### 4.2 痛点 2：引用焦虑 (Citation Anxiety)

#### 现象描述

```
用户的真实声音：

"最怕的就是引用出错，导师看到会很生气。"

"用 AI 生成的内容，我不敢直接用，万一引用是编造的怎么办？"

"花了 2 小时核对 AI 写的一段话，结果发现有 3 处引用对不上。
 还不如我自己写。"

"有时候不确定这个观点是我自己想的还是某篇论文说的，
 不标引用怕抄袭，标了又怕标错。"
```

#### 心理机制分析

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      引用焦虑的心理机制                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│                     学术诚信的高压环境                                   │
│                            │                                            │
│              ┌─────────────┼─────────────┐                             │
│              ▼             ▼             ▼                             │
│         引用出错      被指抄袭      学术不端                            │
│              │             │             │                             │
│              ▼             ▼             ▼                             │
│         导师批评      论文撤稿      学位取消                            │
│              │             │             │                             │
│              └─────────────┼─────────────┘                             │
│                            ▼                                            │
│                       高度恐惧                                          │
│                            │                                            │
│                            ▼                                            │
│              ┌─────────────────────────────┐                           │
│              │  行为表现：                  │                           │
│              │  • 不敢下笔                  │                           │
│              │  • 过度核查                  │                           │
│              │  • 拒绝使用 AI 工具          │                           │
│              └─────────────────────────────┘                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### ManuScript 解决方案

| 问题 | 解决方案 | 原理 |
|------|---------|------|
| 担心引用出错 | **强制引用标注** | 每句话都带 [Doc_ID:Page] |
| 担心 AI 编造 | **封闭域检索** | 只能引用用户已有的文献 |
| 核对成本高 | **一键溯源** | 点击引用立即显示原文 |

### 4.3 痛点 3：结构性卡顿 (Structural Block)

#### 现象描述

```
用户的真实声音：

"打开 Word，光标在那闪，我盯着它发呆了一个小时。"

"知道要写什么，但就是不知道第一句话怎么开头。"

"感觉脑子里有很多想法，但就是组织不起来。"

"写完第一段感觉特别难，后面就顺了。但每次开头都要挣扎很久。"
```

#### 心理机制分析

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    结构性卡顿的心理机制                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   "空白画布恐惧" (Blank Canvas Syndrome)                                │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │                                                                 │  │
│   │    空白文档                                                     │  │
│   │       │                                                         │  │
│   │       ▼                                                         │  │
│   │    无限可能性 → 决策瘫痪（不知道选哪个方向）                    │  │
│   │       │                                                         │  │
│   │       ▼                                                         │  │
│   │    完美主义 → "第一句必须写好" → 压力 → 拖延                   │  │
│   │       │                                                         │  │
│   │       ▼                                                         │  │
│   │    自我怀疑 → "我是不是不适合做研究"                           │  │
│   │                                                                 │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   对比：有草稿可以修改 vs 从零开始创作                                  │
│                                                                         │
│   修改草稿：任务明确，心理负担小                                        │
│   从零创作：任务模糊，心理负担大                                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### ManuScript 解决方案

| 问题 | 解决方案 | 原理 |
|------|---------|------|
| 不知道写什么 | **先展示素材** | 用户先看到"原料"，再组装 |
| 不知道怎么开头 | **AI 生成初稿** | 给用户一个可修改的起点 |
| 决策瘫痪 | **角色转换** | 从"创作者"变成"审核者" |

---

### 4.4 痛点优先级排序

基于影响范围和痛苦程度，排序如下：

```
痛点优先级 = 影响范围 × 痛苦程度 × 现有方案缺失度

┌─────────────────────────────────────────────────────────────────────────┐
│  痛点                    影响范围    痛苦程度    缺失度    优先级得分   │
├─────────────────────────────────────────────────────────────────────────┤
│  1. 引用焦虑              ★★★★★      ★★★★★      ★★★★☆      95         │
│  2. 认知过载              ★★★★★      ★★★★☆      ★★★★☆      80         │
│  3. 结构性卡顿            ★★★★☆      ★★★☆☆      ★★★☆☆      48         │
└─────────────────────────────────────────────────────────────────────────┘

结论：引用焦虑是最核心的痛点，必须优先解决
```

### 5.2 MVP 功能范围

| 优先级 | 功能 | MVP | 理由 |
|--------|------|:---:|------|

## 7. 调研结论与产品启示

### 7.1 核心发现

| 发现 | 产品启示 |
|------|---------|
| **引用焦虑是最大痛点** | "信任"比"效率"更重要，必须优先保证引用可靠 |
| **用户不需要 AI 代写** | 要做"认知脚手架"，不是"代笔工具" |
| **素材组织比信息获取更难** | 用户不缺文献，缺的是"组织和调用"能力 |
| **空白文档是写作杀手** | 提供"可修改的起点"比"从零创作"更有价值 |

### 7.2 产品设计原则

基于用户调研，确立以下设计原则：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     ManuScript 产品设计原则                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 【信任优先】                                                        │
│     每一句生成的内容都必须可追溯                                        │
│     宁可功能简单，也不能让用户担心                                      │
│                                                                         │
│  2. 【降维打击】                                                        │
│     把 50 篇论文的复杂度降维到"这一节需要的 5-10 个片段"               │
│     用户一次只需要处理一个 Section                                      │
│                                                                         │
│  3. 【审核者而非创作者】                                                │
│     用户的角色是"确认和修改"，不是"从零创作"                          │
│     系统先展示素材，用户确认后再生成                                    │
│                                                                         │
│  4. 【封闭域保障】                                                      │
│     只使用用户自己的文献库，不从网络获取                                │
│     "不可能引用不存在的文献"是核心卖点                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
