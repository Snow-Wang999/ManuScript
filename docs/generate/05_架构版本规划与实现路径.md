## v0.1：最小可运行原型（Minimal Viable Prototype）

### 设计理念
> 用最少的代码验证核心假设：用户是否需要"基于文献生成草稿"这个功能？

### 架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                         v0.1 架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户输入                                                      │
│   (题目 + 章节标题)                                              │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    单一 Prompt                           │  │
│   │  System: 你是学术写作助手...                             │  │
│   │  User: 基于以下文献片段，写关于{章节}的内容               │  │
│   │  Context: {RAGFlow 返回的 Chunks}                        │  │
│   └───────────────────────────┬─────────────────────────────┘  │
│                               │                                 │
│                               ▼                                 │
│                          生成草稿                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 技术栈
| 组件 | 选择 | 说明 |
|------|------|------|
| 前端 | Gradio | 3个输入框 + 1个输出框 |
| 后端 | 纯 Python 脚本 | 无框架，50-100 行代码 |
| RAG | RAGFlow API | 直接调用 /retrieval |
| LLM | OpenAI API | 单次调用 |

### 核心代码结构
```python
# v0.1 核心逻辑（约 50 行）
def generate_draft(topic: str, section_title: str, api_key: str):
    # 1. 调用 RAGFlow 检索
    chunks = ragflow_search(section_title, api_key)

    # 2. 拼接 Prompt
    prompt = f"""
    基于以下文献片段，撰写关于"{section_title}"的学术段落。
    要求：每句话标注来源 [Doc_ID:Page]

    文献片段：
    {format_chunks(chunks)}
    """

    # 3. 调用 LLM
    response = openai.chat(prompt)
    return response
```

### 预期发现的问题

- [ ] 引用标注经常出错或遗漏
- [ ] 生成内容可能偏离主题
- [ ] 无法处理复杂的章节结构

### 实现检查清单

### 核心代码结构
```python
# v0.2 核心逻辑

class BasicPipeline:
    def __init__(self, ragflow_key, openai_key):
        self.ragflow = RAGFlowClient(ragflow_key)
        self.llm = OpenAIClient(openai_key)

    def process_section(self, section: dict) -> str:
        # Step 1: Query 优化
        queries = self.generate_queries(section['title'])

        # Step 2: 检索
        chunks = self.ragflow.search(queries)

        # Step 3: 生成草稿
        draft = self.generate_draft(section, chunks)

        # Step 4: 引用格式化
        formatted = self.format_citations(draft, chunks)

        return formatted

    def generate_queries(self, title: str) -> List[str]:
        prompt = f"将以下章节标题转换为3-5个学术检索关键词：{title}"
        return self.llm.chat(prompt).split('\n')
```

### 核心代码结构
```python
# v1.0 Agent 链架构

from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass
class AgentContext:
    """Agent 间传递的上下文"""
    section: dict
    outline: dict
    plan: dict = None
    queries: list = None
    chunks: list = None
    ranked_chunks: list = None
    draft: str = None
    verification: dict = None

class BaseAgent(ABC):
    @abstractmethod
    async def process(self, ctx: AgentContext) -> AgentContext:
        pass

class SectionPlannerAgent(BaseAgent):
    async def process(self, ctx: AgentContext) -> AgentContext:
        prompt = f"""
        分析以下章节的写作意图：
        - 章节标题：{ctx.section['title']}
        - 父级章节：{ctx.section.get('parent', '无')}
        - 论文题目：{ctx.outline['title']}

        输出：1) 写作意图 2) 需要涵盖的要点 3) 建议的检索方向
        """
        ctx.plan = await self.llm.chat(prompt)
        return ctx

class QueryGeneratorAgent(BaseAgent):
    async def process(self, ctx: AgentContext) -> AgentContext:
        prompt = f"""
        基于以下写作计划，生成检索关键词：
        {ctx.plan}

        要求：生成 3-5 个检索词，考虑同义词和相关概念
        """
        ctx.queries = await self.llm.chat(prompt)
        return ctx

# ... 其他 Agent 类似

class MaterialAssemblyPipeline:
    def __init__(self):
        self.agents = [
            SectionPlannerAgent(),
            QueryGeneratorAgent(),
            RetrievalAgent(),
            ChunkRankerAgent(),
            DraftWriterAgent(),
            VerifierAgent(),
        ]

    async def process_section(self, section: dict, outline: dict) -> str:
        ctx = AgentContext(section=section, outline=outline)

        for agent in self.agents:
            ctx = await agent.process(ctx)

        return ctx.draft
```

### 预期发现的问题

- [ ] 各 Agent 之间的 Prompt 需要精心设计
- [ ] Verifier 可能过于严格或过于宽松

### 实现检查清单
- [ ] 6 个 Agent 类实现

### 核心代码结构
```python
# v2.0 Orchestrator-Worker 架构

class Orchestrator:
    def __init__(self):
        self.workers = {
            'simple': SimpleWorker(),
            'complex': ComplexWorker(),
            'review': ReviewWorker(),
        }

    async def analyze_complexity(self, section: dict) -> str:
        """分析章节复杂度"""
        prompt = f"""
        分析以下章节的复杂度：
        - 标题：{section['title']}
        - 类型判断：
          - simple: 单一概念介绍
          - complex: 多概念对比/方法描述
          - review: 综述/文献回顾
        """
        return await self.llm.chat(prompt)

    async def process_section(self, section: dict) -> str:
        complexity = await self.analyze_complexity(section)
        worker = self.workers[complexity]
        return await worker.process(section)

    async def process_outline(self, outline: dict) -> dict:
        """并行处理多个章节"""
        sections = self.flatten_sections(outline)

        # 并行执行
        tasks = [self.process_section(s) for s in sections]
        results = await asyncio.gather(*tasks)

        return dict(zip([s['id'] for s in sections], results))
```

### 预期发现的问题
- [ ] 复杂度判断不准确
- [ ] 并行处理可能导致 Token 超限
- [ ] Worker 之间缺乏信息共享（后面章节不知道前面写了什么）
- [ ] 调试难度增加

### 实现检查清单
